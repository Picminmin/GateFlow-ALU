「GateFlow ALU」という名前のローカルWebアプリを作成してください。

目的:
ALUで行われる基本的な演算が、論理ゲートレベルでどのように処理されるかをリアルタイムで可視化する教育用アプリを作りたいです。
バスや配線を流れる信号は、光る点として表示し、ユーザーが信号の流れを目で追えるようにしてください。

技術スタック:
- React
- TypeScript
- Vite
- 回路描画はSVG
- 最初の版ではバックエンド不要

MVPの範囲:
- まずは1bit full adderのみ実装
- 入力: A, B, Cin
- 出力: Sum, Cout
- 論理ゲートと配線を表示
- 信号伝搬を、配線上を動く点でアニメーション表示
- ゲート出力が変化したら、そのゲートの見た目も変化
- 再生、一時停止、リセット、1ステップ送りを実装
- アニメーション速度の調整を実装

重要機能:
回路実装モードを複数用意してください。
1. Primitive mode: 基本ゲートを明示的に並べた素朴な構成
2. Optimized mode: より少ないゲート数、または浅い段数で同じ結果を出す構成
3. 余裕があれば後で NOR-only mode を追加できる設計にする

モード切替時には、見た目だけでなく実際の回路グラフ自体を切り替えてください。
表示されるゲート数、配線、信号伝搬経路が変わる必要があります。

UI要件:
- 左パネル: 入力設定とモード選択
- 中央: 回路可視化
- 右パネル: 選択したゲートの詳細（ゲート種別、現在の入力、現在の出力、遅延）
- 下部: 信号変化イベントのログ表示

アニメーション要件:
- 信号は光る点として配線上を流れる
- logic 1 の配線は強調表示
- 内部は離散イベントシミュレーションで実装
- 各ゲートに小さな伝搬遅延を持たせる
- イベント単位でステップ実行できること

設計要件:
- 回路定義、シミュレーションエンジン、描画を分離
- 回路は型付きグラフデータで表現
- ノード、エッジ、ゲート評価を再利用可能にする
- 後から half adder, 4bit ripple-carry adder, 簡易ALUに拡張しやすい構造にする

成果物:
1. Vite + React + TypeScript の動くアプリ
2. 明確なフォルダ構成
3. 回路定義、シミュレータ、SVG描画、操作パネルを分割した実装
4. 実行方法を書いた README
5. 読みやすいコード

最初はMVPだけ作成してください。
最初からALU全体を実装しないでください。
まず1bit full adderを完成させ、その後4bit版を追加しやすい土台を作ってください。
